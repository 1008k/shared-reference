# GitMCP

この文書は、外部のGitHubリポジトリやドキュメントをAIエージェントに参照させたい場合の任意拡張メモです。仕様は `docs/project-spec.md`、セットアップと運用は `README.md` を参照。

## 向いている場面
- OSSライブラリの実装やREADMEを、その場で確認しながら判断したい
- 依存先の最新の使い方や移行状況を、GitHub上の正本に寄せて確認したい
- 自分のリポジトリだけでは不足する周辺文脈を、限定的に補いたい

## 利点
- 外部リポジトリのコード、設定、文書を、会話中に直接参照しやすい
- ライブラリ導入前の調査や比較が速くなる
- docs-first な段階でも、外部依存候補の実態を確認しやすい

## 注意点
- 外部通信を伴うため、採用時は通信先と用途を明記する
- GitHub上の情報は更新されるため、判断根拠に使う日時を残す
- 社内限定情報や機密情報を扱う用途には、そのまま使わない

## このテンプレートでの扱い
- 標準では有効化しない
- 外部ライブラリ調査が継続的に必要な案件でのみ導入候補にする
- 採用した場合は、`README.md` または別紙に対象範囲と運用ルールを追記する
