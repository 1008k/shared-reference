# Context Tools Trial

この文書は、Serena / Context7 / Headroom / Context Mode などのコンテキスト削減系ツールを試すための補助メモです。
恒久ルールではなく、効果が見えたものだけを `AGENTS.md` や各ルール文書へ戻します。

## 位置づけ
- 目的は、圧縮より先に取得する情報量を減らし、必要な根拠だけを読むこと
- 既存のプロジェクト固有ルール、仕様、設計判断を置き換えない
- ツールが利用できない環境では、通常の `rg`、定義確認、差分確認、公式ドキュメント参照で代替する

## 試したい使い分け
- `Serena`: コード探索で、ファイル全体を読む前にシンボル、定義元、参照元、呼び出し元を確認する
- `Context7`: フレームワーク、ライブラリ、API、CLI、クラウドサービスの仕様確認で、最新の公式ドキュメントを参照する
- `Headroom`: 大量ログ、長文書、多数ファイルの結果を扱う必要がある場合にだけ、重要情報を失わない範囲で要約する
- `Context Mode`: 大きなツール出力、セッション継続、圧縮前後の作業文脈保持を、MCP と hooks で補助する

## Context Mode の試験導入観点
- 公式リポジトリ: <https://github.com/mksglu/context-mode>
- Codex CLI では MCP と hooks の組み合わせとして導入候補にする
- 導入前に、Node.js の要件、Codex 側の hooks / plugin hooks の有効化、保存先、外部通信、hook 承認の扱いを確認する
- プロジェクト直下の `AGENTS.md` を外部テンプレートで上書きしない。必要な指示だけ既存ルール体系に合わせて統合する
- まずはグローバルまたは個人環境で試し、このリポジトリ固有の永続設定が必要になってから `.codex/config.toml` などを検討する

## 探索の既定順
1. シンボル、定義、参照、呼び出し元を確認する
2. 関係するファイルだけを読む
3. 必要ならディレクトリ構造を確認する
4. リポジトリ全体の探索は、理由がある場合に限る

## Headroom 等で失ってはいけない情報
- API 定義
- 関数シグネチャ
- 型定義
- 設定値
- スタックトレース
- エラーメッセージ
- テスト失敗の期待値と実際値

## 見直し条件
- 同じ効果が複数回確認できたら、短い恒久ルールとして `AGENTS.md` や `docs/rules-coding.md` へ戻す
- 実際には使わない、または判断を重くするだけならこのメモを削除または縮約する
