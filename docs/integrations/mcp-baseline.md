# MCP baseline

この文書は、汎用プロジェクトで最初に検討しやすいMCPの基本セットを整理する補助文書です。仕様は `docs/project-spec.md`、セットアップと運用は `README.md` を参照。

## 位置づけ
- 目的は、AIエージェントがローカルファイル、Git履歴、GitHub上の変更文脈に安全にアクセスできる最小構成を定義すること
- 対象は、Webアプリ、CLI、ライブラリ、社内ツールなど、種類を問わず再利用しやすい基本連携
- この文書は既定構成を強制しない。採用する場合だけ、対象エージェントと運用責任を明記する

## 推奨する基本セット
- `filesystem`: 作業ディレクトリ内の文書、設定、コードを読むための基本アクセス
- `git`: 変更差分、履歴、ブランチ状況を読み取るための基本アクセス
- `github`: PR、issue、レビューコメント、CI結果を扱う必要がある場合の追加候補

## 採用条件
- 人とAIエージェントの両方が、同じ正本を参照しながら判断したい
- ローカルの文書や差分を、会話の外から都度読み直せる状態にしたい
- GitHub連携が必要な場合でも、閲覧権限や運用責任を説明できる

## 既定で含めないもの
- 永続メモリ、外部検索、外部SaaS連携は、この基本セットには含めない
- バックグラウンドワーカーや常駐プロセスを前提にする構成は、まず外す
- 権限が広い連携や監査境界が曖昧な連携は、個別判断に分ける

## 文書化するときの観点
- 何のために有効化するか
- どの範囲にアクセスできるか
- 外部通信の有無
- 失敗時に作業をどう継続するか
