# OpenMemory Lite

この文書は、汎用テンプレートで扱いやすい軽量ローカルメモリ拡張の考え方を整理する補助文書です。仕様は `docs/project-spec.md`、セットアップと運用は `README.md` を参照。

## 位置づけ
- ここでいう `Lite` は、ダッシュボードや複数サービス連携を前提にせず、ローカル永続化を主目的にした運用を指す
- 対話履歴や簡単な記憶を残したいが、重い基盤はまだ不要な段階を想定する

## 向いている場面
- 同じプロジェクトでAIとの対話を継続し、過去の判断を後から再利用したい
- 永続メモリを試したいが、最初から大きな依存や複雑な運用は避けたい
- ローカルPC上で閉じた形で運用したい

## 利点
- 導入時の負荷が比較的軽い
- docs-first の初期段階でも、判断履歴の散逸を減らしやすい
- 高機能メモリ基盤を入れる前の踏み台として使いやすい

## 注意点
- 保存先のパスとバックアップ方針を決めないと、ローカル永続化の価値が下がる
- メモリの粒度や保存条件を決めないと、ノイズが増えやすい
- チーム共有や高度な検索が必要になった時点で、別の基盤に移行する可能性がある

## MemPalaceとの整理
- `MemPalace` は高機能で魅力があるが、Python、ChromaDB、追加運用を前提にしやすい
- このテンプレートでは、まず軽量メモリを任意候補として扱い、`MemPalace` は要件が明確な案件で個別採用する
- 標準候補にする場合は、成熟度、保守性、依存の重さを再評価してから判断する
