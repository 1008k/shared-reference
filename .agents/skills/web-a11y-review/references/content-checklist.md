# Content Accessibility Checklist

Use this for landing pages, articles, Markdown, CMS entries, and WordPress posts.

## Headings

- [ ] The heading hierarchy reflects the article structure.
- [ ] The first heading names the page or article topic.
- [ ] Headings are not used only for visual emphasis.
- [ ] Long pages have scannable section headings.

## Links

- [ ] Link text identifies the destination or action.
- [ ] Repeated links with the same text go to the same destination, or the text distinguishes them.
- [ ] External links, downloads, PDFs, and new-window behavior are clear when relevant.
- [ ] Raw URLs are avoided unless the URL itself is useful content.

## Images And Alt Text

- [ ] Informational images have concise alt text.
- [ ] Decorative images have empty alt text.
- [ ] Complex charts or diagrams have nearby text summaries.
- [ ] Screenshots explain the relevant state, not every visual detail.
- [ ] Images of text are avoided for important content.

## Tables

- [ ] Tables are used for data, not layout.
- [ ] Header cells are clear.
- [ ] Captions or surrounding text explain what the table is for.
- [ ] Wide tables remain usable on mobile.

## Lists And Emphasis

- [ ] Lists use real list markup.
- [ ] Bold, color, or size is not the only way to indicate importance.
- [ ] Important warnings include text labels such as "注意" or "必須".

## Video, Audio, And Embedded Services

- [ ] Video/audio has captions, transcripts, or equivalent text when it carries important information.
- [ ] Autoplay is avoided or can be paused.
- [ ] Embedded maps, forms, and third-party widgets have accessible names or nearby alternatives.

## Forms And CTAs In Content

- [ ] CTA labels describe the result: "資料をダウンロード", "相談を予約".
- [ ] Form instructions state required information before submission.
- [ ] Error or completion states are understandable without color alone.

## Japanese Content Clarity

- [ ] Sentences are as short as the audience allows.
- [ ] Specialized terms are explained on first use.
- [ ] Date, time, price, deadline, and eligibility conditions are explicit.
- [ ] Ruby, kana, or simpler wording is considered when the audience needs it.
- [ ] Machine-translated or generated text is checked for unnatural instructions.

## Markdown And WordPress

- [ ] Markdown headings start at the correct level for the renderer.
- [ ] Shortcodes, blocks, or custom HTML do not break heading order or landmarks.
- [ ] WordPress block styles preserve front-end and editor readability.
- [ ] Uploaded media has meaningful alt text or is marked decorative.
- [ ] PDF links include enough context about the file and purpose.
